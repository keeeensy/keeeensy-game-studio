<h1 align="center">🧊 Gem Hunter 3D</h1>
<p align="center"><i>3D лабиринт от первого лица — трёхмерная версия Gem Hunter 2D</i></p>

---

Проходи процедурно-генерируемые 3D-лабиринты, собирай все сферы, чтобы открыть выход, и выживай среди врагов на 10 уровнях. Сделано на C++17 и raylib 5.5.

## ✨ Возможности

- **10 процедурных уровней** — BFS-связанные лабиринты, 8×8 → 26×26
- **3 типа врагов** — Scout (быстрый, хрупкий), Soldier (сбалансированный), Brute (медленный, живучий) — открываются по ходу игры
- **Спринт** — Shift для скорости x1.7
- **3 жизни** — контактный урон с 2-секундной неуязвимостью
- **Миникарта** — в правом верхнем углу: исследованные зоны, игрок, выход, сферы
- **Процедурный звук** — все звуки синтезируются на лету, без внешних файлов
- **Настройки** — ползунки громкости и чувствительности мыши
- **Greedy mesh merge** — стены объединяются в меши для производительности рендеринга

## 🎮 Управление

| Клавиша | Действие |
|---------|----------|
| WASD / Стрелки | Движение |
| Shift | Спринт (x1.7) |
| Мышь | Осмотреться |
| ESC | Вернуться в меню |
| ENTER | Пропустить интро / подтвердить |

## 🧩 Прогрессия уровней

| Уровень | Размер | Сфер | Врагов |
|---------|--------|------|--------|
| 1       | 8×8    | 4    | 0      |
| 3       | 12×12  | 8    | 2 (Scout) |
| 5       | 16×16  | 10   | 3      |
| 7       | 20×20  | 12   | 4      |
| 10      | 26×26  | 12   | 5      |

- Scout (ур. 2+), Soldier (ур. 4+), Brute (ур. 6+)

## 📦 Сборка

### Требования

- Windows
- CMake 3.15+
- MinGW g++ (или MSVC)
- raylib (скачивается CMake автоматически)

### Шаги

```bash
cmake -B build -G "MinGW Makefiles"
cmake --build build
```

Или просто запусти `build.bat`. Исполняемый файл копируется в корень как `maze_run_3d.exe`.

### Валидация

```bash
maze_run_3d.exe --validate
```

Запускает генерацию всех 10 уровней и сообщает о проблемах с поиском пути.
